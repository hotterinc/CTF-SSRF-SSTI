# SOLVE

Краткая инструкция по решению обеих задач.

## SSRF
Цель: через форму URL получить доступ к внутреннему сервису на `127.0.0.1:9000`.

1) Открой SSRF-страницу:\n   - http://127.0.0.1:8000/ssrf
2) Для проверки сначала отправь любой внешний URL (например, `https://example.com`).\n3) Теперь отправь внутренний адрес:\n   - `http://127.0.0.1:9000`
4) В ответе появится подсказка вроде `admin, now go to secret`.\n5) Запроси секретный эндпоинт:\n   - `http://127.0.0.1:9000/secret`\n6) Ответ содержит секретную информацию.

## SSTI
Цель: использовать рендеринг комментариев для чтения секретного значения из контекста шаблона.

1) Открой SSTI-приложение:\n   - http://127.0.0.1:8000/ssti
2) Зарегистрируй двух пользователей (например, `alice` и `bob`).\n3) Войди как `alice`.\n4) Перейди на профиль `bob` и оставь комментарий с Jinja2-выражением:\n   - `{{ secret_key }}`
5) Перезагрузи страницу профиля `bob` и увидишь отрендеренное значение.

Примеры альтернативных полезных пейлоадов:
- `{{ current_user }}`
- `{{ users.keys() }}`

Примечание: `secret_key` добавлен в контекст рендеринга в `server.py`.
